# Чек-лист деплоя

## Supabase

- [ ] Создан отдельный production-проект.
- [ ] Выполнен `supabase/migrations/0001_schema.sql`.
- [ ] Выполнен `supabase/seed.sql` либо подготовлены реальные данные.
- [ ] RLS включён на всех бизнес-таблицах.
- [ ] Получены Project URL, anon key и service role key.
- [ ] Заполнена локальная `.env.local`.
- [ ] Выполнен `npm run seed:auth` для демонстрационных аккаунтов либо добавлены реальные сотрудники.
- [ ] Проверена связь `profiles.auth_user_id` с `auth.users.id`.

## Vercel

- [ ] Репозиторий импортирован как Next.js проект.
- [ ] Node.js Runtime — 20 или новее.
- [ ] Добавлены `NEXT_PUBLIC_SUPABASE_URL` и `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
- [ ] Добавлен server-only `SUPABASE_SERVICE_ROLE_KEY`.
- [ ] Добавлен `NEXT_PUBLIC_EMPLOYEE_EMAIL_DOMAIN`.
- [ ] Добавлен точный production URL в `NEXT_PUBLIC_SITE_URL`.
- [ ] Установлен `NEXT_PUBLIC_DEMO_MODE=false`.
- [ ] Production deployment завершён без ошибок.

## Совпадение доменов

После первого деплоя скопируйте конечный домен Vercel, например:

```text
https://bionit-v-dele.vercel.app
```

Укажите его без завершающего `/`:

```text
Vercel:   NEXT_PUBLIC_SITE_URL=https://bionit-v-dele.vercel.app
Supabase: Authentication → URL Configuration → Site URL
```

Эти значения должны совпадать. После подключения собственного домена замените оба значения и сделайте redeploy.
